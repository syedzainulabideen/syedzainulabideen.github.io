SOLID Core Design Concept

SOLID - Open Close Principle (OCP)

OCP states that software entities (such as classes, modules, functions, etc.) should be open for extension but closed for modification. In simple words it means that once a entity is created and working as intended, its internal code should not be modified to add new features or behaviors. So how can then we add features or update the functionalities? And the answer is by extending the existing entities with by Subclassing, Extensions or Protocols/Interfaces etc.
