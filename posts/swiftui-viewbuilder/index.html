SwiftUI - View Builders

@ViewBuilder is a parameter attribute that constructs child views from closures in a resulable manner. @ViewBuilder is one of the possible result builders provided in Swift language. Every element in SwiftUI, uses @ViewBuilder to build it’s child Views. Like HStack:

struct ContentView: View {
     var body: some View {
         HStack{
             Text("First Text")
             Text("Second Text")
         }
     }
 }

And if we check the defination of HStack, We can see the it gets content using the @ViewBuilder. and this @ViewBuilder clousure handles the construction of HStack’s child views.

@inlinable public init(
     alignment: VerticalAlignment = .center,
     spacing: CGFloat? = nil,
     @ViewBuilder content: () -> Content
 )

Another important thing to notice is that we have pass exactly two child views in HStack, so Complier will be calling the @ViewBuilder’s static buildBlock function that exactly accept two parameters. and the @ViewBuilder’s buildBlock have other initiliztion options as well that take one to ten parameters, that’s the reason Views in SwiftUI can have upmost 10 child views.

@available(iOS 13.0, OSX 10.15, tvOS 13.0, watchOS 6.0, *)
 extension ViewBuilder {
     public static func buildBlock<C0, C1>(_ c0: C0, _ c1: C1) -> TupleView<(C0, C1)> where C0 : View, C1 : View
 }

As @ViewBuilder can only be applied to a parameter, function, or computed property only. Let’s take an example to see how @ViewBuilder help us to create more useable views using a parameter.

struct FlexibleHeader<Content:View>: View {
     var content:Content
     init(@ViewBuilder content: () -> Content) {
         self.content = content()
     }
     var body: some View {
         HStack {
             Text("Flexible Header")
             Spacer()
             content
         }
     }
 }

 struct ContentView: View {
     var body: some View {
         VStack {
             FlexibleHeader {
                 Text("Hello")
             }
             
             FlexibleHeader {
                 Image(systemName: "heart")
             }
         }
     }
 }

In above example, we can see that using the @ViewBuilder attribute in function parameter we have the flexibility to choice our right view in FlexibleHeader. @ViewBuilder can also very helpful while we have to return different types of Views from a function.

struct ContentView: View {
     var body: some View {
         VStack {
             self.getMyView(option: true)
         }
     }
     
     @ViewBuilder
     func getMyView(option:Bool) -> some View {
         if option {
             OneView()
         }
         else {
             TwoView()
         }
     }
 }

 struct OneView: View {
     var body: some View {
         HStack {
             
         }
     }
 }

 struct TwoView: View {
     var body: some View {
         Button("") {
             
         }
     }
 }
